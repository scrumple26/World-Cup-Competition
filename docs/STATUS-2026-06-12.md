### 1. WC standings now update from real results — FIXED
API-Football labels the group tables **`"Group Stage - Group A"`**, but
`toGroupStandings` filtered on `/^Group [A-Z]$/`, so **every row was dropped**
and standings stayed frozen at 0 (this — not an "empty API" — was the real
cause; yesterday's note misdiagnosed it). `normalizeGroupLabel` now canonicalizes
the label to `"Group A"` and drops the junk aggregate ranking table. Verified:
`/api/wc/standings` returns 12 groups with live results; `wcStandings` persisted
(summed played = 4). Unit tests added in `src/lib/wcMap.test.ts`.

### 2. Pundit recaps — root cause was a Firestore crash, now FIXED + backfilled
The "failproof" feed generation was deployed but produced **zero** entries.
Cause: **Firestore throws on `undefined` by default**, and 4 of 18 profiles have
no `logoUrl`; that `undefined` nested inside each feed entry's `perUser` array
made every `feedEntries.set()` throw — swallowed by the per-match catch, which
silently zeroed **all** recap + full-time/goal tweet generation. Fixed once,
globally, with `firestore.settings({ ignoreUndefinedProperties: true })` in
`src/lib/firebase/admin.ts`. Both played games now have **AI pundit recaps**
(4 commentary lines each, real stats/scorers/perfect-pickers). This was never a
cron-cadence problem.

### 3. Goal tweets — added (live drip) + failproof backfill
- **Live drip:** `/api/buzz` (`runGoals`) posts one fan tweet per goal as it
  happens, idempotent on deterministic ids `tweets/goal_<fixtureId>_<i>`. Single
  calls spread across 5-min ticks ⇒ AI-written.
- **Backfill net:** `feedGen` re-checks the same ids on every sync, so any goal
  the live cron missed is filled in with no duplicates.
- `generateGoalTweets` (AI + varied template fallback) in `src/lib/social.ts`.
- Backfilled all 5 goals from the two completed games.
