### Pick deadline model
- Single soft+hard deadline `PICK_DEADLINE_ISO` in `src/lib/config.ts`
  (currently **Sun Jun 14 2026, 11:59 PM Central**). Change that one value to move it.
- Everyone locks in their whole bracket **all at once** (no per-pick auto-save).
- Soft: scoring skips any game that kicked off **before** the player's lock-in
  time (`lockedAt`), so late lock-ins score 0 on games already underway.
- Hard: after the deadline, server-side lockout in the lock-in / predictions /
  profile routes (no new accounts, no lock-ins, no pick edits; admin bypasses).
- No random auto-fill and no auto-lock — not locked in by the deadline = 0s.

### Scoring is working (verified on prod)
- Opener scored correctly: perfect 2–0 picks = 3 pts (e.g. Todd, Nolan, NotNolan,
  Andy, Austin). Outcome-only = 1, outcome + one exact digit = 1.5.
- The earlier "everything shows 0" was stale/not-yet-recomputed data + client cache.

### Charts
- `ScoreDoc.history` is now a **per-game** cumulative series (game 1 = first
  completed WC match), rebuilt deterministically each recompute from FINAL
  results only. X axis shows the bare game number; tooltip says "Game N".
- Group charts (cumulative points, projected final standings, win probability)
  are collapsible accordions.

## ⚠️ Action required / known issues

1. **Backfill the opener's recap/pundit/tweets.** The opener had **no feed entry**
   (generation was failing). Generation is now failproof + self-healing. To
   backfill immediately: **Admin → "⟳ Sync results now"** once. It will create the
   missing recap, pundit piece, and full-time result tweets (the next 3-hour cron
   would also do it). Live per-goal tweets are **not** a feature yet — tweets are
   pre-match, half-time, and full-time (the FT tweets mention the goals/scorers).

2. **Firestore quota (root cause of most failures).** Recompute and feed
   generation are read-heavy. On the free Spark tier (50k reads/day) they fail on
   match days, which is what blanked the feed/pundit. **Confirm the project is on
   Blaze with a billing account linked.** Generation now degrades gracefully, but
   it can't beat a hard-exhausted quota.

3. **Vercel cron cadence.** `vercel.json` schedules `/api/sync` every 3h and
   `/api/buzz` every 5 min. On the **Hobby** plan these are throttled (≈daily), so
   score/content freshness relies on the post-match auto-sync (fires when someone
   has the app open as a match ends). Consider **Vercel Pro** for true sub-daily
   crons, or accept the post-match-sync path.

4. **Gemini key for AI pundit/tweets.** AI uses `GEMINI_API_KEY` (model
   `gemini-2.5-flash`). Without it (or on quota) everything falls back to
   templates — content still posts, just not AI-written. Confirm the key is set in
   Vercel if you want AI copy.
